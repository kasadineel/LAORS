LAORS Coding Rules

Goal:
Ship LAORS Core and the Stocker module MVP.

Rules:
- Do not add new libraries unless necessary.
- Keep code simple.
- Follow the current folder structure.
- Always check that the app still builds.
- Do not hide modules only in the UI, enforce them in the backend too.

Definition of Done:
- App deploys
- Organizations can enable the Stocker module
- Stocker pages are inaccessible if the module is disabled
